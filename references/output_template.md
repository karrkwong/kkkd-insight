# kkkd-insight 输出模板

适用目标：输出正式书面化的《产品判断书》或《方向判断书》，用于立项、优先级、范围收敛或策略定义。

## 标题要求

- 文档标题统一采用：`<命题名称>_机会洞察`
- 一级、二级、三级、四级章节之间都加入一条 Markdown 分割线：`---`
- 二级章节标题可直接写结论，格式优先采用：`<章节名>：<一句话结论>`

## 固定结构

```md
# <命题名称>_机会洞察

## 一、核心判断
- 工作模式：从零研究 / 材料收敛
- 方向定位：
- 优先级：
- 是否成立：
- 成立前提：
- 不成立边界：

## 二、看行业
### 看行业：<一句话结论>
- 赛道阶段判断
- 底层变量表：变量 / 当前状态 / 对命题影响 / 判断
- 行业含义：3 条以内

## 三、看用户
### 看用户：<一句话结论>
- 用户分层：人群 / 核心任务 / 高价值场景
- VOC 证据概览：样本量口径 / 来源平台 / 当前可用性判断
- 高频痛点表
- 高频满意点表
- 用户场景旅程表：按主场景从起点到终点展示关键节点，每个节点写明用户动作 / 问题或痛点 / 用户期望
- Mermaid 用户场景旅程图：用流程图形式表达同一条主场景链路
- 标杆标准：定性标准 + 定量阈值
- 用户结论

## 四、看竞争
### 看竞争：<一句话结论>
- 竞品地图：竞品 / 解法概述 / 功能界面截图（至少 2 张） / 是否达到标杆 / 优势 / 主要短板
- 竞争盲区：盲区 / 用户后果 / 对我方意义
- 竞争结论

## 五、看机会
### 看机会：<一句话结论>
- 仅保留 3 个以内战略机会点：机会点 / 成立原因 / 对应用户问题 / 竞争缺口 / 破局潜力
- 明确列出不应进入首期范围的伪机会
- 机会结论

## 六、看自己
### 看自己：<一句话结论>
- 自身能力映射：能力维度 / 当前状态假设 / 对机会点支撑 / 主要短板
- 自身结论

## 七、定控制点
### 定控制点：<一句话结论>
- 3-4 个控制点：控制点 / 形成方式 / 防守价值 / 建设难度
- 控制点结论

## 八、定目标
### 定目标：<一句话结论>
- 分阶段目标表：阶段 / 目标定义 / 必须兑现能力 / 优先级 / 成功标志
- 首发人群
- 首发场景

## 九、定策略
### 定策略：<一句话结论>
- 产品策略
- 体验策略
- 研发策略
- 区域或渠道策略（如适用）
- 市场策略

## 十、最终结论
- 是否做：
- 先做什么：
- 不做什么：
- 以什么控制点防守：
- 当前最大不确定性：

## 十一、分析输入说明
- 输入材料
- 证据边界
- 推断边界

## 十二、备注项
- 推测项
- 假设项
- 证据缺口
- 后续补证方向
```

## 强制要求

- 每一节必须输出结论句，不得只有材料堆叠。
- `看机会` 必须收敛，不得把所有可能方向都保留为“可做”。
- `最终结论` 必须回答“做 / 不做 / 缩做 / 延后”。
- 正文只写已证实内容；推测、假设、证据缺口与后续补证方向统一进入`备注项`。
- 不得使用“不是A，而是B”“不取决于A，而取决于B”“真实价值点不在A，而在B”这类对仗式判断句。
