# K5K3D

面向产品判断书的中文写作 skill。它用于围绕某个功能、场景、技术或产品方向，按 `5看3定` 框架输出正式中文的机会判断文档。

仓库内的 skill 适合两类任务：

- 从零启动某个命题的研究，并最终收敛成一份可执行的产品判断书
- 基于已有研究材料、VOC 摘要、竞品分析或会议纪要，快速完成判断收束

## 它解决什么问题

很多产品分析最终会退化成三种低效结果：

- 材料很多，但没有明确判断
- 竞品罗列很多，但没有取舍
- 结论看起来完整，但证据边界和执行动作不清楚

`K5K3D` 的目标是把分析文档稳定约束为：

- 先给结论
- 再展开 `5看`
- 最后落到 `3定`
- 所有推测、假设、证据缺口统一沉到附录

## 适用场景

- 分析某个 AI 功能是否值得进入产品路线
- 围绕一个新场景输出方向判断书
- 在立项前做范围收敛、优先级判断、MVP 路线选择
- 对已有研究材料做结构化决策收束

## 不适用场景

- 通用 PRD
- 普通行业综述
- 纯竞品清单
- 没有证据支撑的泛战略写作

## 核心框架

### 5看

- `看行业`
- `看用户`
- `看竞争`
- `看机会`
- `看自己`

### 3定

- `定控制点`
- `定目标`
- `定策略`

## 输出特点

- 使用正式中文
- 标题统一为 `<命题名称>_机会洞察`
- 第一部分必须是 `执行摘要`
- 正文只保留已证实内容
- 推测、假设、证据缺口统一写入附录 `备注项`
- 至少包含一张对比表或判断矩阵
- `看用户` 强制包含：
  - VOC
  - 场景链路
  - 基准标准
  - 用户场景旅程表
  - Mermaid 场景旅程流程图
- `看竞争` 强制包含核心对标对象截图，每家不少于 2 张

## 工作方式

这个 skill 支持两种模式：

### 1. 从零研究模式

当用户只给命题，没有足够材料时：

- 先补齐行业、用户、竞争的最小必要研究
- 再输出判断书

### 2. 材料收敛模式

当用户已经给出研究材料、VOC 摘要或竞品分析时：

- 先做证据分层
- 再做判断收束

## 文档结构

标准结构如下：

1. `执行摘要`
2. `5看`
3. `3定`
4. `结论与下一步`
5. `附录`

仓库中的 [references/output_template.md](./references/output_template.md) 给出了固定模板。

## 证据与写作约束

- 优先使用一手证据和用户已提供材料
- 不把建议样本量写成已完成样本量
- 不把推测写成事实
- 不使用营销式 AI 表述
- 不使用修辞化对仗句，例如：
  - “不是 A，而是 B”
  - “不取决于 A，而取决于 B”
  - “真实价值点不在 A，而在 B”

详细口径见 [references/research_standards.md](./references/research_standards.md)。

## 适合的输出风格

这个 skill 更适合产出：

- 判断明确
- 结构稳定
- 可执行
- 对证据边界敏感

而不是：

- 资料堆叠
- 泛泛建议
- 漫谈式产品分析
