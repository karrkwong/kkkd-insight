# k5k3d

## 何时使用

- 用户要求从零分析一个功能、场景、技术或方向，并输出完整产品判断书。
- 用户已经提供研究材料、竞品分析、VOC 摘要、会议纪要或内部判断输入，需要收敛为一份正式文档。
- 用户明确要求使用`5看3定`、产品判断书、方向判断、机会收敛、立项判断或路线选择文档。

## 不适用场景

- 不用于通用 PRD、需求规格说明、普通行业综述、纯竞品罗列或周报摘要。
- 当用户没有提供足够输入，且又明确限制不能补做研究时，应先说明缺口，再输出边界化判断或补证建议。
- 不在没有证据、也不允许研究的前提下伪造完整判断书。

## 输出契约

- 使用正式书面语，不使用聊天式、讲解式、口语化表达。
- 文档标题统一写为`<命题名称>_机会洞察`。
- 第一部分必须是`执行摘要`，直接写结论、依据、建议动作。
- 主体固定为`5看`与`3定`，不得跳项；信息不足时不在正文中展开提示，统一放入附录`备注项`。
- 结论必须可执行。
- 至少包含一张对比表或判断矩阵，支持快速扫描。
- 一级、二级、三级、四级章节之间都必须加入分割线，提升可读性与扫描效率。
- 二级章节标题可直接承载结论，例如`看行业：该赛道仍有结构性空间`、`看机会：竞品尚未满足核心诉求`。
- 涉及 AI 时，仅描述能力边界、触发机制、输入输出、Agent-Skill 分工与成本约束，不写模型宣传或泛化表述。
- 正文中不得把推测写成事实；凡属推测、假设或证据缺口，统一放入附录`备注项`，不在正文额外标注提示。

## 工作流

1. 界定判断对象。
   - 明确对象是功能、场景、技术还是产品方向。
   - 明确文档目标是立项、优先级、MVP 收敛、路线选择还是资源投入判断。
   - 明确当前是`从零研究模式`还是`材料收敛模式`。
   - `从零研究模式`：先补齐证据，再进入判断书撰写。
   - `材料收敛模式`：先盘点已有输入的证据等级与缺口，再进入判断书撰写。
2. 组织研究输入。
   - `从零研究模式`：按`看行业 -> 看用户 -> 看竞争 -> 看机会 -> 看自己`补齐核心材料。
   - `材料收敛模式`：复核用户提供的材料，标出可直接采用、需要验证、明显缺口三类输入。
   - VOC、竞品、标杆标准、自身能力映射四类输入必须至少形成工作底稿；达不到完整度时保留缺口标签。
3. 先写执行摘要。
   - 先给结论，再给原因，再给建议动作。
   - 结论不回避取舍，可直接写`建议推进`、`建议限定范围试点`、`建议暂不推进`。
4. 完成`5看`。
   - `看行业`：判断行业阶段、结构性变化、关键驱动因素、时间窗口。
   - `看用户`：必须包含 VOC；明确目标用户、场景链路、现有替代方案、基准体验标准。若无逐条 VOC，明确写为主题级证据。
   - `看用户`必须同时包含用户场景旅程表与 Mermaid 流程图示意，按主场景从起点到终点拆出关键节点，并在每个节点下说明用户动作、问题或痛点、用户期望，保证前后链路逻辑通顺。
   - `看竞争`：比较直接竞争、替代方案、默认解决路径，不只罗列功能；对每一家核心对标竞品至少收集`2`张功能界面截图，用于校准真实交互与功能落点。
   - `看机会`：指出价值集中点、未被满足的需求、切入口、成立条件。
   - `看自己`：审视自身资产、能力边界、组织条件、成本结构、交付风险。
5. 完成`3定`。
   - `定控制点`：明确必须控制的变量、关键依赖、验收口径、风险边界。
   - `定目标`：给出阶段目标、衡量指标、非目标。
   - `定策略`：给出产品路线、范围取舍、推进顺序、资源投放方式。
6. 收束为决策文档。
   - 用一句话再次确认方向。
   - 列出下一步动作、验证事项、暂缓事项。

## 证据政策

- 优先使用用户已提供的研究材料与一手证据；若任务要求从零启动，则主动补做外部研究。
- 不把“建议至少采集多少样本”写成“已经完成多少样本”。
- 当用户要求从零分析，或判断依赖显著时效信息时，应补做外部研究并标注来源。
- 若研究范围过大，先收窄到关键命题、关键人群、关键竞品，再补证，不做无边界横扫。
- 强判断来自清晰推演，不来自伪装的高确定性。
- 正文只写已证实结论；推测、假设、证据缺口与补证方向统一放入附录`备注项`。

#### 2.5 看自己：<一句话结论>

## 强制要求

- `看用户`必须出现 VOC、场景链路、基准标准三项。
- `看用户`必须包含一张从用户数据中提炼出的场景旅程表，以及一张对应的 Mermaid 场景旅程流程图，覆盖主场景的关键节点、节点痛点与节点期望。
- 结论必须包含明确倾向，不能只做信息整理。
- 所有假设必须仅在附录`备注项`中显式收录，不能把猜测写成事实。
- 正文仅保留已证实内容；若存在假设、推测或证据缺口，统一收纳至附录`备注项`。
- 若用户已经给出文风约束，以用户要求优先。
- 从零研究模式下，必须主动补做行业、用户、竞争三类最小必要研究，不得把“缺输入”当成默认停止条件。
- `看竞争`至少覆盖核心对标对象的功能界面截图，每家不少于`2`张。

## 禁止事项

- 不要写成行业综述、材料摘抄或竞品清单。
- 不要使用“简单来说”“你可以这样理解”“本质上就是”等聊天式表达。
- 不要把 AI 写成万能能力，不要使用营销化措辞。
- 不要回避取舍，不要只写“都有价值”“可以进一步观察”一类弱判断。
- 不要遗漏控制点、非目标、风险边界。
- 严禁使用“不是A，而是B”“不取决于A，而取决于B”“真实价值点不在A，而在B”这类对仗式判断句。
- 严禁出现类似“竞争目标不应是‘谁更会写’，而应是‘谁更适合发出去’”的写法。

## 示例触发

- `分析语音搜索能力的产品规划，输出一份判断书。`
- `围绕企业知识库问答场景做一份5看3定产品分析。`
- `评估某项 AI 技术是否适合纳入当前产品路线，并给出产品判断。`
